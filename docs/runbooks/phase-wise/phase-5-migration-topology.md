# Phase 5 — Migration and Advanced Topology

## Objective
Demonstrate migration and advanced topology patterns.

## Topics
Traditional HA/HS, Modern controller in separate Kubernetes cluster, external Jenkinsfile/shared library pattern, OSS Traditional to CloudBees Modern migration, Team Controller to Managed Controller migration.

## Steps
1. Document or configure Traditional HA/HS.
2. Demonstrate multi-cluster Modern topology concept.
3. Configure job using properties file and separate Jenkinsfile/shared library repo.
4. Migrate sample job from OSS Traditional to CloudBees CI Modern.
5. Migrate job from Team Controller to Managed Controller.

## Evidence
Topology diagrams, source/target job configs, migration notes, successful target builds, console outputs, known gaps.

## Exit Criteria
At least one migration path demonstrated, topology constraints documented, migration evidence captured, demo completed.
